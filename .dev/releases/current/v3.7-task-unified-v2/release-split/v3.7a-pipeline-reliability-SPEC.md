## 1. Problem Statement

The Sprint CLI pipeline has five compounding defects that silently degrade every production sprint run. Three are data-integrity bugs in the Path A execution path (`_run_task_subprocess()` at `executor.py:1064-1092`), one is a data-loss bug in the pipeline process layer (`process.py:114`), and a naming collision across 21+ files creates ambiguity between the deprecated `sc:task-unified` and the canonical `sc:task` command. Together, these defects render the TurnLedger reimbursement system, anti-instinct gate evaluation, per-phase turn/token tracking, and checkpoint enforcement all inoperative for production sprints that use properly structured tasklists.

Separately, the Sprint CLI lacks any checkpoint enforcement mechanism. Agents may complete all T-numbered tasks but skip `### Checkpoint:` sections, resulting in missing verification reports that operators only discover hours later through manual inspection. The three-layer defense model (prompt prevention, post-phase detection gate, manifest and recovery) is entirely absent.

This release fixes the five bugs, adds targeted prompt enrichment to Path A (~280 tokens per task), implements three waves of checkpoint enforcement, and resolves the naming collision -- establishing the infrastructure and data contracts that Release 2 (v3.7b TUI v2) depends on.

## 2. Solution Overview

This release makes four categories of change:

1. **Data integrity fixes** (PA-04, PA-05, PA-06, MA-03): Wire `count_turns_from_output()` into `_run_task_subprocess()`, set `TaskResult.output_path`, change `gate_rollout_mode` default to `"shadow"`, and change output file mode from `"w"` to `"a"` (append). These are 1-3 LOC changes each that activate dormant infrastructure (TurnLedger reimbursement, anti-instinct gate evaluation, multi-task NDJSON preservation).

2. **Prompt enrichment** (PA-01, PA-02, PA-03): Extract per-task markdown blocks from phase files via `_extract_task_block()`, add explicit scope boundaries ("Execute ONLY task X. STOP when complete."), and inject a shared sprint context header. Adds ~280 tokens per task subprocess prompt, replacing the current 3-line minimal prompt.

3. **Naming consolidation** (N1-N12): Delete the deprecated `task.md`, rename `task-unified.md` to `task.md`, rename the skill directory, update all cross-references across ~21 files, and run `make sync-dev`.

4. **Checkpoint enforcement** (T01-T03): Three waves implementing prompt-level prevention (Wave 1), post-phase detection gate in shadow mode (Wave 2), and manifest/recovery/CLI infrastructure (Wave 3).

**New function** (`config.py`, alongside `parse_tasklist`):
```python
def extract_task_block(phase_file: Path, task_id: str) -> str:
    """Extract the raw markdown block for a single task from a phase file.

    Returns the heading line and all content up to the next task heading
    or EOF. Returns empty string if task_id is not found.
    """
    content = phase_file.read_text(encoding="utf-8", errors="replace")
    headings = list(_TASK_HEADING_RE.finditer(content))
    for i, match in enumerate(headings):
        if match.group(1) == task_id:
            block_start = match.start()
            block_end = headings[i + 1].start() if i + 1 < len(headings) else len(content)
            return content[block_start:block_end].rstrip()
    return ""
```

**After prompt** (`executor.py:1064-1068`, modified):

### FR-37A.04: Implement extract_token_usage() (DM-04)

**Description**: Add `extract_token_usage(output_path: Path) -> tuple[int, int]` to `monitor.py` alongside the existing `count_turns_from_output()`. Parses NDJSON `usage` blocks to sum `input_tokens` and `output_tokens`.

**New function** (`monitor.py`, after `count_turns_from_output`):
```python
_USAGE_PATTERN = re.compile(
    r'"usage"\s*:\s*\{[^}]*"input_tokens"\s*:\s*(\d+)[^}]*"output_tokens"\s*:\s*(\d+)'
)

def extract_token_usage(output_path: Path) -> tuple[int, int]:
    """Extract cumulative token usage from subprocess NDJSON output.

    Parses "usage" blocks in stream-json events to sum input_tokens
    and output_tokens across all events.

    Returns:
        (tokens_in, tokens_out) tuple. Returns (0, 0) if file missing or unparseable.
    """
    try:
        content = output_path.read_text(errors="replace")
    except (FileNotFoundError, OSError):
        return (0, 0)

    tokens_in = 0
    tokens_out = 0
    for line in content.splitlines():
        m = _USAGE_PATTERN.search(line)
        if m:
            tokens_in += int(m.group(1))
            tokens_out += int(m.group(2))
    return (tokens_in, tokens_out)
```

**Acceptance Criteria**:
- [ ] `extract_token_usage()` correctly sums input/output tokens from multi-line NDJSON
- [ ] Returns `(0, 0)` for missing, empty, or unparseable files
- [ ] Handles NDJSON with zero usage blocks gracefully
- [ ] Handles NDJSON with multiple usage blocks per turn (accumulates all)

**Verification Contract**: Unit test with mock NDJSON containing 3 usage blocks verifies correct sum. Separate test with malformed JSON verifies `(0, 0)` return.

**Impact Analysis**: ~25 LOC added to `monitor.py`. Pure function, no side effects. Positioned alongside `count_turns_from_output()` using the same file-reading pattern.

**Commit Boundary**: Independent commit. DM-05 depends on this.

**Rollback Plan**: Delete the function. No callers outside this release depend on it until DM-05 wires it.

**Dependencies**: None.

**New shared function** (in `process.py` or a shared utility):
```python
def build_sprint_context_header(config: SprintConfig, phase: Phase) -> str:
    """Build the ## Sprint Context markdown header for task prompts."""
    pn = phase.number
    sprint_name = getattr(config, "release_name", None) or config.release_dir.name
    total_phases = len(config.phases) if config.phases else pn
    artifact_root = config.release_dir / "artifacts"
    results_dir = config.results_dir
    prior_phase_dirs = (
        [config.release_dir / f"phase-{i}" for i in range(1, pn)]
        if pn > 1 else []
    )
    lines = [
        "## Sprint Context",
        f"- Sprint: {sprint_name}",
        f"- Phase: {pn} of {total_phases}",
        f"- Artifact root: {artifact_root}",
        f"- Results directory: {results_dir}",
    ]
    if prior_phase_dirs:
        lines.append(
            "- Prior-phase directories: "
            + ", ".join(str(d) for d in prior_phase_dirs)
        )
    return "\n".join(lines)
```

## 9. Migration & Rollout

- **Breaking changes**: Naming consolidation is a breaking change for code that references `/sc:task-unified` directly. All known references are updated in N1-N12. The Sprint CLI prompt and cleanup_audit prompts are updated.

- **Backwards compatibility**: `gate_rollout_mode` default changes from `"off"` to `"shadow"`. Shadow mode has zero behavioral impact (metrics only). Users can pass `--gate-mode off` to restore old behavior. `checkpoint_gate_mode` defaults to `"shadow"` with the same safety guarantee.

- **Rollback plan**: Each phase has an independent rollback:
  - Phase 0: Revert `executor.py` changes (turns, output_path). Revert `process.py:114` (`"a"` to `"w"`).
  - Phase 0.5: Revert prompt enrichment lines. Revert `models.py:329` (`"shadow"` to `"off"`).
  - Phase 1: `git revert` the naming commit. Re-run `make sync-dev`.
  - Phase 2-4: Delete `checkpoints.py`. Revert `executor.py` checkpoint hooks. Revert `models.py` enum/config additions.

### Migration Scenarios

**Scenario 1: User has custom scripts referencing `/sc:task-unified`**
