## 1. Problem Statement

The sprint TUI (`tui.py`) provides a minimal dashboard that shows phase-level progress but lacks task-level telemetry, error visibility, post-phase summaries, and token consumption data. Under Path A (the production execution path handling every properly structured tasklist), the TUI receives degraded data: 9 of 12 `MonitorState` fields remain at defaults because `OutputMonitor` is never started for Path A. The operator sees `0 B` output, `+0.0 B/s` growth, `0` files changed, and a perpetual `"waiting..."` status. After a sprint completes, there is no automated summary of what happened across phases -- the operator must manually inspect NDJSON logs.

This release transforms the TUI from a basic status board into a rich telemetry dashboard by: (1) implementing the PhaseAccumulator adapter pattern (Strategy B) to populate MonitorState fields from accumulated TaskResult data, (2) adding 7 rendering features (F1-F4, F6-F7, plus F8 summarizer), (3) building a post-phase summary pipeline using Haiku narratives, (4) integrating tmux 3-pane layout for live summary display, and (5) generating release retrospectives.

## 2. Solution Overview

This release implements Strategy B (synthetic MonitorState population via PhaseAccumulator) as the architectural adapter between Path A's per-task subprocess model and the TUI's MonitorState-based rendering contract. The PhaseAccumulator accumulates telemetry across sequential task completions and projects a MonitorState snapshot for each TUI update cycle. This approach was selected over Strategy A (per-task OutputMonitor) because the blocking `proc.wait()` at `executor.py:1086-1087` makes real-time TUI updates structurally unreachable without non-blocking subprocess changes not in scope (Debate 1, convergence 87.5%).

The solution adds two new modules (`summarizer.py`, `retrospective.py`), modifies four existing modules (`models.py`, `tui.py`, `executor.py`, `tmux.py`), and creates the PhaseAccumulator class as the central data bridge. All TUI rendering changes consume MonitorState as the single data contract (Debate 3 ruling), with the adapter fixing the `stall_status` false alarm by setting `last_event_time = time.monotonic()` on every synthetic update.

### 2.1 Key Design Decisions

| Decision | Choice | Alternatives Considered | Rationale |
|----------|--------|------------------------|-----------|
| Monitor strategy | Strategy B: Synthetic MonitorState via PhaseAccumulator | Strategy A: Per-task OutputMonitor lifecycle (103 LOC, 6 downstream sites); Strategy C: Hybrid (A+B) | Blocking subprocess (`executor.py:1086-1087`) makes real-time TUI unreachable. B delivers same between-task cadence at ~48 LOC with zero downstream adaptations. Debate 1: B wins 6-1-1, convergence 87.5%. |
| TUI data contract | Shared MonitorState interface with adapter | Path-specific data objects (`PathAData`/`PathBData` with separate rendering) | MonitorState IS the existing interface (`tui.py:107-108`). Adding a second type doubles rendering paths for no structural benefit. Adapter fixes `stall_status` (P2). Debate 3: Shared wins 4-2-2, convergence 87.5%. |
| stall_status fix | Adapter sets `last_event_time = monotonic()` | Modify stall_status property to accept path flag; Add is_path_a field to MonitorState | Adapter-level fix is minimal (1 LOC), preserves MonitorState's Path B semantics, and is transparent to TUI. |
| PhaseSummarizer input | Dual input: `list[TaskResult]` (Path A) OR NDJSON path (Path B) | NDJSON only (Path B); TaskResult only (breaks Path B); Convert TaskResult to synthetic NDJSON | Path A has structured per-task data (superior for status/errors). Path B has continuous NDJSON. Dual input serves both without degradation. |
| F5 LLM Context Lines | Deprioritized for Path A | Implement with per-task prompt preview; Implement Path B only in R2 | Per-task prompt is 3 lines after PA-01 enrichment. Preview value is negligible. Becomes useful only with non-blocking subprocess (v3.8). MERGED Section 1.4 ruling. |
| Summary generation | Background daemon thread (SummaryWorker) | Inline blocking after phase; Separate process | Background thread avoids blocking sprint execution. Thread pool with Lock guards `_summaries` dict. 30s timeout on Haiku call prevents hangs. |
| Tmux integration | 3-pane layout (TUI 50% + summary 25% + tail 25%) | 2-pane (TUI + summary); Full-screen TUI with overlay | 3-pane provides simultaneous access to dashboard, latest summary, and raw output. Tail pane index shifts from `:0.1` to `:0.2`. |

    last_task_id: str = ""
    _last_event_time: float = field(default_factory=time.monotonic)

    def pre_task(self, task_id: str) -> None:
        """Mark task as starting. Sets last_event_time for stall suppression."""
        self.last_task_id = task_id
        self._last_event_time = time.monotonic()

    def harvest(self, task_id: str, result: "TaskResult") -> None:
        """Accumulate completed task telemetry."""
        self.total_output_bytes += result.output_bytes
        self.total_turns += result.turns_consumed
        self.total_tokens_in += result.tokens_in
        self.total_tokens_out += result.tokens_out
        self.tasks_completed += 1
        self._last_event_time = time.monotonic()

        status = "PASS" if result.status.is_success else f"FAIL({result.exit_code})"
        self.activity_log.append((time.monotonic(), task_id, status))
        if result.status.is_failure:
            self.errors.append((task_id, "subprocess", f"exit code {result.exit_code}"))

    def to_monitor_state(self) -> "MonitorState":
        """Project accumulated data into MonitorState for TUI consumption.

        CRITICAL: Sets last_event_time = monotonic() to suppress false stall alarms.
        Without this fix, stall_status shows "STALLED" for any task >120s.
        """
        state = MonitorState()
        state.output_bytes = self.total_output_bytes
        state.events_received = self.tasks_completed
        state.last_event_time = time.monotonic()  # STALL FIX
        state.last_task_id = self.last_task_id
        state.turns = self.total_turns
        state.tokens_in = self.total_tokens_in
        state.tokens_out = self.total_tokens_out
        state.total_tasks_in_phase = self.tasks_total
        state.completed_task_estimate = self.tasks_completed
        state.activity_log = list(self.activity_log)
        state.errors = list(self.errors[-10:])  # cap at 10
        state.stall_seconds = 0.0
        return state

## 5. Interface Contracts

### 5.1 PhaseAccumulator Interface

```python
class PhaseAccumulator:
    def __init__(self, tasks_total: int = 0): ...
    def pre_task(self, task_id: str) -> None: ...
    def harvest(self, task_id: str, result: TaskResult) -> None: ...
    def to_monitor_state(self) -> MonitorState: ...
```

**Contract**: `to_monitor_state()` MUST set `last_event_time = time.monotonic()` on every call. This suppresses the `stall_status` false alarm at `models.py:522-535` which shows "STALLED" after 120s without events.

### 5.2 PhaseSummarizer Interface

```python
class PhaseSummarizer:
    def summarize(
        self,
        phase: Phase,
        *,
        task_results: list[TaskResult] | None = None,
        ndjson_path: Path | None = None,
        config: SprintConfig | None = None,
    ) -> PhaseSummary: ...
```

**Contract**: Exactly one of `task_results` or `ndjson_path` must be provided. `ValueError` if neither or both.

### 5.3 SummaryWorker Interface

```python
class SummaryWorker:
    def __init__(self, config: SprintConfig): ...
    def submit(self, phase: Phase, task_results: list[TaskResult] | None = None,
               ndjson_path: Path | None = None, config: SprintConfig | None = None) -> None: ...
    def get_summary(self, phase_number: int) -> PhaseSummary | None: ...
    def shutdown(self, timeout: float = 60.0) -> None: ...
```

**Contract**: `submit()` is non-blocking. `_summaries` dict guarded by `threading.Lock`. All exceptions caught internally -- summary failure never affects sprint execution.

### 5.4 Haiku Narrative Pipeline

```python
def generate_haiku_narrative(prompt: str, *, timeout: float = 30.0) -> str: ...
```

**Contract**:
- Checks `shutil.which("claude")` before attempting subprocess call
- Strips `CLAUDECODE` and `CLAUDE_CODE_ENTRYPOINT` from subprocess environment
- Command: `claude --print --model claude-haiku-4-5 --max-turns 1 --dangerously-skip-permissions`
- Returns empty string on any failure (timeout, missing binary, subprocess error)
- Never raises exceptions to caller

**Contract**: Steps 1-2 are synchronous. Step 3 is non-blocking. Steps 4-6 execute regardless of 1-3 outcomes.
