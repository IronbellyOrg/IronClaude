# DEV-007: Fix roadmap test file paths to match spec and actual directory structure

## Deviation Summary

The roadmap specifies test files under `tests/cli/sprint/` which does not exist. The spec
correctly uses `tests/sprint/`, matching the established project convention. Additionally,
the roadmap omits `test_classifiers.py` from its files-created table despite the spec
listing it as a new file with 5 dedicated unit tests.

## Validation Evidence

- **`tests/sprint/` EXISTS** -- contains 20+ test files (test_executor.py, test_process.py, test_monitor.py, etc.)
- **`tests/cli/sprint/` DOES NOT EXIST** -- `tests/cli/` directory is entirely absent from the project
- **`tests/cli/` DOES NOT EXIST** -- no `cli` subdirectory under `tests/` at all
- **Spec references (correct)**: `tests/sprint/test_preflight.py`, `tests/sprint/test_classifiers.py`
- **Roadmap references (incorrect)**: `tests/cli/sprint/test_preflight.py`; omits `test_classifiers.py`

The roadmap also references the incorrect path in its Success Criteria table (SC-004, SC-005):
```
uv run pytest tests/cli/sprint/test_preflight.py -m unit
uv run pytest tests/cli/sprint/test_preflight.py -m integration
```

## Required Changes
