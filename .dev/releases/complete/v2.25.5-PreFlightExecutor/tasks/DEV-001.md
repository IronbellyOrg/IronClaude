# DEV-001: Place `execute_preflight_phases()` in `executor.py`, Not New `preflight.py`

## Deviation Summary

The roadmap creates a new file `src/superclaude/cli/sprint/preflight.py` for `execute_preflight_phases()` core logic. The spec explicitly mandates that this function be added to the existing `src/superclaude/cli/sprint/executor.py`, which already contains `execute_sprint()` and all phase orchestration logic.

**Confirmed findings:**
- `execute_sprint()` lives in `executor.py` (line 491), not `process.py` as the roadmap claims
- No `preflight.py` file exists in the codebase today
- `executor.py` already contains all phase execution infrastructure: `execute_phase_tasks()`, `_run_task_subprocess()`, `AggregatedPhaseReport`, `_determine_phase_status()`, `setup_isolation()`, and `_write_executor_result_file()`
- Placing preflight logic in `executor.py` maintains colocation with the functions it must integrate with (`execute_sprint()`, `AggregatedPhaseReport.to_markdown()`, `_determine_phase_status()`)

**This also resolves DEV-002:** The roadmap's claim that `execute_sprint()` lives in `process.py` is incorrect. The spec correctly identifies `executor.py` as the integration target.

## Implementation Requirement

Update the roadmap to place `execute_preflight_phases()` inside `src/superclaude/cli/sprint/executor.py` instead of creating a new `preflight.py` file. During implementation:

1. **Add `execute_preflight_phases(config: SprintConfig) -> list[PhaseResult]`** to `executor.py`
   - Position it after `execute_phase_tasks()` (after line ~448) and before `execute_sprint()` (line 491)
   - It filters `config.active_phases` where `execution_mode == "python"`
   - For each phase: parse tasks, execute commands via `subprocess.run(shell=False, capture_output=True, timeout=120)`
   - Build `TaskResult` per task, `PhaseResult` per phase using existing dataclasses from `models.py`
   - Write result files via `AggregatedPhaseReport.to_markdown()` (already in `executor.py`)

2. **Modify `execute_sprint()`** (line 491 in `executor.py`) to:
   - Call `execute_preflight_phases(config)` before the main phase loop (before line 530)
   - Skip phases where `execution_mode == "python"` in the main loop
   - Skip phases where `execution_mode == "skip"` with `PhaseStatus.SKIPPED`
   - Merge preflight results into `sprint_result.phase_results`

3. **Do NOT create `src/superclaude/cli/sprint/preflight.py`**
