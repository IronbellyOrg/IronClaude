# DEV-011: Align Roadmap File Lists with Spec (New Files Table)

## Deviation Summary

The spec lists 3 new files: `classifiers.py`, `tests/sprint/test_preflight.py`, `tests/sprint/test_classifiers.py`. The roadmap lists 3 different files: `classifiers.py`, `preflight.py`, `tests/cli/sprint/test_preflight.py`. Two discrepancies exist:

1. **`preflight.py` appears in roadmap but not in spec.** Already resolved by DEV-001: `execute_preflight_phases()` belongs in `executor.py`, not a new `preflight.py`. No action needed here beyond confirming DEV-001 correction removes `preflight.py` from the "Files created" table.

2. **`test_classifiers.py` appears in spec but not in roadmap.** The roadmap's Phase 2 narrative describes classifier tests but does not list `test_classifiers.py` as a created file. The codebase convention in `tests/sprint/` is one test file per module (e.g., `test_executor.py`, `test_config.py`, `test_models.py`, `test_monitor.py`). Since `classifiers.py` is a new module with its own responsibility, a separate `test_classifiers.py` is the correct approach.

Note: The test path issue (`tests/sprint/` vs `tests/cli/sprint/`) is covered by DEV-007.

**Confirmed findings:**
- No `tests/cli/` directory exists; all sprint tests live under `tests/sprint/`
- Existing test convention is one-file-per-module: `test_executor.py`, `test_config.py`, `test_models.py`, etc.
- `classifiers.py` is a standalone module (classifier registry + built-in classifiers) warranting its own test file
- Roadmap Phase 2 describes 3 classifier-specific tests that belong in a dedicated file

## Implementation Requirement

Update the roadmap's Resource Requirements "Files created" table and Phase 2 test section to align with the spec:

1. **Remove `preflight.py`** from the "Files created" table (per DEV-001)
2. **Add `tests/sprint/test_classifiers.py`** to the "Files created" table with purpose: "Unit tests for classifier registry and built-in classifiers"
3. **Correct `tests/cli/sprint/test_preflight.py`** to `tests/sprint/test_preflight.py` (per DEV-007, but relevant to this table)
4. **Update Phase 2 test section** to reference `test_classifiers.py` as the target file for classifier unit tests
