# DEV-006: Add config.py parser extensions to roadmap modified files table

## Context

The spec-fidelity report identified that the roadmap's "Code Areas Impacted / Files modified" table omits `config.py` as a target for parser extension work. The table lists `config.py (or model file)` only for dataclass changes (`Phase`, `TaskEntry`), but does not account for the parser functions that also live in `config.py` and require modification.

## Deviation

**Spec requirement:**
- FR-001 (FR-PREFLIGHT.1): `discover_phases()` in `config.py` must be extended to parse the `Execution Mode` column.
- FR-003 (FR-PREFLIGHT.3): `parse_tasklist()` in `config.py` must be extended to extract `**Command:**` and `| Classifier |` fields.

**Roadmap gap:**
The "Files modified" table (roadmap lines 237-243) contains:

| File | Change Type |
|------|------------|
| `config.py (or model file)` | Extend `Phase`, `TaskEntry` dataclasses |
| `process.py` | Add preflight call + skip logic |
| `PhaseStatus` enum location | Add `PREFLIGHT_PASS` |

No row covers parser extensions in `config.py`.

## Validation Evidence

Both functions confirmed present in `src/superclaude/cli/sprint/config.py`:
- `discover_phases()` at line 27
- `parse_tasklist()` at line 196

The roadmap's Phase 1 narrative (lines 30-58) correctly describes the parser work but the resource requirements table does not reflect it. This creates a traceability gap: an implementer consulting the files-modified table would not know `config.py` requires parser changes beyond dataclass additions.

## Task

Add a dedicated row to the roadmap's "Files modified" table for parser extension work in `config.py`.

### Acceptance Criteria

- [ ] The roadmap "Files modified" table contains a row for `src/superclaude/cli/sprint/config.py` with change type: "Extend `discover_phases()` to parse Execution Mode column; extend `parse_tasklist()` to extract `**Command:**` and `| Classifier |`"
- [ ] The existing `config.py (or model file)` row for dataclass changes is disambiguated (remove parenthetical ambiguity per DEV-005, or keep if DEV-005 is addressed separately)
- [ ] The Phase 1 narrative and the files-modified table are consistent -- every file touched in Phase 1 steps appears in the table
