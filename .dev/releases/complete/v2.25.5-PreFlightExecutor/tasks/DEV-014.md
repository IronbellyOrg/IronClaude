# DEV-014: Add test_classifiers.py to roadmap files-created table

## Context

The spec-fidelity report identified that the spec lists `tests/sprint/test_classifiers.py` as a new file (under section 4.1, "New Files"), but the roadmap's "Files created" resource requirements table does not include it. The roadmap's Phase 2 narrative describes three classifier-specific unit tests but does not list a dedicated test file for them.

## Deviation

**Spec requirement:**
- Section 4.1 New Files: `tests/sprint/test_classifiers.py` -- Unit tests for classifier functions
- FR-004 (FR-PREFLIGHT.4): A new module `classifiers.py` must provide the classifier registry

**Roadmap gap:**
The "Files created" table (roadmap lines 245-251) contains:

| File | Purpose |
|------|---------|
| `src/superclaude/cli/sprint/classifiers.py` | Classifier registry + `empirical_gate_v1` |
| `src/superclaude/cli/sprint/preflight.py` | `execute_preflight_phases()` core logic |
| `tests/cli/sprint/test_preflight.py` | Unit + integration tests |

No row for `test_classifiers.py`.

## Validation Evidence

**1. Project convention supports a separate file.** The existing codebase follows a 1:1 source-to-test mapping:
- `executor.py` (38,683 bytes) -> `test_executor.py` (1,292 lines)
- `process.py` (12,436 bytes) -> `test_process.py` (460 lines)
- `config.py` (8,914 bytes) -> `test_config.py` (360 lines)
- `models.py` (19,373 bytes) -> `test_models.py` (889 lines)

A new `classifiers.py` module warrants its own `test_classifiers.py` to maintain this convention.

**2. Phase 2 defines standalone classifier tests.** The roadmap's Phase 2 test section (lines 73-75) describes three tests that are classifier-specific and independent of preflight execution:
- Unit test `empirical_gate_v1` with known pass/fail inputs
- Unit test `KeyError` on missing classifier name
- Unit test exception handling in classifier invocation

These tests exercise `classifiers.py` in isolation and do not depend on `preflight.py`.

**3. Neither file exists yet.** No `test_classifiers.py` or `test_preflight.py` exists in the codebase. No `classifiers.py` source module exists either. All are new files to be created during implementation.

**4. Related deviations.** DEV-007 notes a test path mismatch (`tests/sprint/` vs `tests/cli/sprint/`). DEV-011 notes the file list divergence more broadly. DEV-014 addresses the specific omission from the resource requirements table, which is the implementer's primary reference for file creation.

## Task

Add `test_classifiers.py` to the roadmap's "Files created" resource requirements table.
