# DEV-013: Estimation Format Difference -- T-Shirt Sizes vs LOC Counts

## Deviation Summary

The spec provides LOC-based complexity estimates ("~400 LOC new code + ~200 LOC tests") and specific task counts per phase in Section 10. The roadmap's Timeline Estimates table uses T-shirt sizing ("Small", "Medium", "Small-Medium") for per-phase effort.

**Confirmed findings:**
- The deviation is purely cosmetic. The two estimation approaches serve different purposes: the spec's LOC figures describe total project scope, while the roadmap's T-shirt sizes describe relative per-phase effort.
- The roadmap's Executive Summary (paragraph 2) already includes the spec's LOC figures verbatim: "~400 LOC of new code across 3 new files, ~100 LOC of modifications to 3 existing files, and ~200 LOC of tests."
- No functional requirement, acceptance criterion, architecture decision, or implementation detail is affected by the estimation format.
- Both estimation approaches are valid and complementary. The T-shirt sizes add per-phase granularity that the spec's aggregate LOC figure does not provide.

## Recommended Correction

None required. This is acknowledged as a cosmetic difference with no implementation impact.
