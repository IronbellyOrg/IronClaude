# DEV-010: Add `[preflight]` Annotation to `--dry-run` Output

## Deviation Summary

The spec identifies GAP-02: `--dry-run` should list python-mode phases with a `[preflight]` annotation in `_print_dry_run()`. The roadmap's Open Questions table marks OQ-002 as "Resolved" but no phase, task, deliverable, or test implements this behavior. The current `_print_dry_run()` function in `commands.py` has no awareness of `execution_mode` and displays all phases identically regardless of their execution mode.

**Verdict: The roadmap claims resolution without delivering it. Both a code task and a test are missing.**

## Task

**Target file**: `src/superclaude/cli/sprint/commands.py`

**Function**: `_print_dry_run()` (line 304)

**Implementation**:

Modify the phase display loop in `_print_dry_run()` to append a `[preflight]` tag when `phase.execution_mode == "python"` and a `[skip]` tag when `phase.execution_mode == "skip"`. The annotation makes dry-run output clearly communicate which phases will be handled by the preflight executor versus the Claude subprocess loop.

Current output format per phase:
```
  Phase N: <display_name>
    File: <path>
```

Required output format for python-mode phases:
```
  Phase N: <display_name> [preflight]
    File: <path>
```

Required output format for skip-mode phases:
```
  Phase N: <display_name> [skip]
    File: <path>
```

Claude-mode phases remain unchanged (no annotation).

**Test file**: Same test file designated for preflight tests (per roadmap: `tests/cli/sprint/test_preflight.py`)

**Test**: Assert that `_print_dry_run()` output includes `[preflight]` for python-mode phases, `[skip]` for skip-mode phases, and no annotation for claude-mode phases. Use `click.testing.CliRunner` to capture output.

## Acceptance Criteria

- [ ] `_print_dry_run()` appends `[skip]` tag to display line for phases with `execution_mode == "skip"`
- [ ] Claude-mode phases display with no annotation (backward-compatible)
