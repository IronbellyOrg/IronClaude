## Themes

### 1. PRD/TDD context gap in mid-pipeline steps
The diff (step 3), debate (step 4), and merge (step 6) steps operate without PRD/TDD supplementary context. Steps 1, 2, 5, 7, 8 all received PRD/TDD awareness -- but the adversarial debate that decides which roadmap variant wins is blind to business context.

### 2. YAML frontmatter parsing fragility
The pipeline relies on a regex (`_FRONTMATTER_RE`) for gate validation of every step's output, yet this regex has at least two edge-case failure modes: hyphenated keys and blank lines within frontmatter. The project already declares `pyyaml>=6.0` as a dependency but doesn't use it for gate validation.

### 3. Build/packaging inconsistencies
The `force-include` in pyproject.toml doubles the wheel content, and the Makefile sync/verify targets have portability issues with spaces and hidden files.

### 4. Enum name collision
Two independent `TaskStatus` enums in different modules create a latent wrong-import risk.

## Recommendations

1. **Highest priority**: Fix the PRD/TDD context gap in diff/debate/merge steps (C-63, C-64). These are the steps that decide the final roadmap shape, so operating blind to supplementary context undermines the entire PRD/TDD integration.
2. **Medium priority**: Replace the frontmatter regex with pyyaml parsing (C-65, C-66). The dependency already exists; using it eliminates an entire class of gate false-negatives.
3. **Low priority but easy**: Fix the duplicate step-8 comment (C-62), rename the parallel TaskStatus (C-71), clean up the build config (C-69).
