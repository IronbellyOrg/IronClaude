# PRD Pipeline Integration -- Wire --prd-file Across Roadmap and Tasklist Pipelines

## Task Overview

This task implements `--prd-file` as a supplementary input across both the `superclaude roadmap run` and `superclaude tasklist validate` CLI pipelines. It also wires the existing dead `tdd_file` field on `RoadmapConfig` end-to-end, refactors 4 prompt builders from single-return to base pattern (P1/P2) and adds `prd_file` parameter stubs to 3 more (P3), adds PRD supplementary blocks to 9 prompt builders, stores supplementary file paths in `.roadmap-state.json` for auto-wiring by the tasklist pipeline, enriches tasklist generation (not just validation) with TDD/PRD content, and updates 4 skill/reference layer documents.

The end state enables: `superclaude roadmap run tdd.md --prd-file prd.md` with the tasklist auto-detecting both files from state. The TDD is the primary input (auto-detected). The PRD provides business context -- personas, success metrics, compliance requirements, user journeys -- that the TDD extraction drops (only 5 of 28 PRD sections survive extraction). All changes are additive with `None` defaults for zero backward-compatibility risk.

## Key Objectives

- Add `--prd-file` CLI flag to both `roadmap run` and `tasklist validate` commands
- Wire the dead `tdd_file` field on `RoadmapConfig` through CLI, executor, and prompt builders with a redundancy guard
- Refactor 4 prompt builders (`build_generate_prompt`, `build_score_prompt`, `build_spec_fidelity_prompt`, `build_test_strategy_prompt`) from single-return to base pattern
- Add PRD supplementary blocks to 9 prompt builders (extract, extract_tdd, generate, score, spec-fidelity, test-strategy, plus P3 param stubs for diff/debate/merge)
- Add tasklist fidelity prompt PRD block
- Store `tdd_file` and `prd_file` in `.roadmap-state.json` for downstream auto-wiring
- Implement auto-wire logic in tasklist executor with precedence rules (explicit flag > state > None)
- Enrich tasklist generation prompts with TDD/PRD content blocks
- Update 4 skill/reference documents: extraction-pipeline.md, scoring.md, tasklist SKILL.md, spec-panel.md
- Comprehensive test coverage for 6 interaction scenarios (A-F) across both pipelines
