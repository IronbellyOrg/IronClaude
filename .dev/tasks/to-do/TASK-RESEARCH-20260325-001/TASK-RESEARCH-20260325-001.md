# CLI TDD Integration: Technical Investigation & Research Report

## Task Overview

This task drives a deep technical investigation into what Python CLI files must change to support `superclaude roadmap run <tdd_file>` — accepting a TDD (Technical Design Document) as input instead of a PRD spec. A prior research investigation audited the inference/skill layer and found that TDD sections are poorly captured by the 8-step extraction pipeline, but was unaware of the programmatic Python CLI layer. This task covers that CLI layer end-to-end and produces a structured 10-section research report with verified findings, gap analysis, options, and a precise implementation plan.

The investigation spans three CLI subsystems: `src/superclaude/cli/roadmap/` (executor, prompts, gates, parser, analysis modules), `src/superclaude/cli/tasklist/` (commands, models, validation gate), and `src/superclaude/cli/pipeline/` (generic pipeline layer — expected to require no changes). Six parallel research agents investigate distinct slices. Quality gates (rf-analyst + rf-qa) verify completeness before synthesis. Six parallel synthesis agents produce the 10-section report structure. The assembled report is validated by two QA passes (structural and qualitative) before being presented to the user.

## Key Objectives

- Map all paths through which `spec_file` flows in `executor.py` — both LLM-mediated (embedded in prompt) and programmatic (Python functions reading file directly)
- Audit all prompt builders in `prompts.py` for spec-specific language that would fail on TDD input; identify exact text changes required
- Verify TDD compatibility of all 5 pure-Python analysis modules (spec_parser, fidelity_checker, integration_contracts, fingerprint, obligation_scanner)
- Analyze all 14 gate definitions for TDD compatibility; produce per-gate verdict table
- Identify all CLI extension points needed (flags, model fields) in `roadmap/commands.py` and `tasklist/commands.py`
- Map all 28 TDD sections against the current extract prompt's capture capability; identify missed sections and required extraction instructions
- Synthesize findings into a complete 10-section research report with implementation plan covering specific files, functions, and change descriptions
