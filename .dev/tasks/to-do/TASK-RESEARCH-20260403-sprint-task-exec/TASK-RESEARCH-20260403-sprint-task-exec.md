# Deep Technical Investigation: Sprint Task Execution Pipeline Verification Layers

## Task Overview

This task investigates whether the IronClaude sprint task execution pipeline has per-task verification, QA, or behavioral governance layers that were missed in preliminary analysis, or whether identified gaps are real. Preliminary analysis found that Path A (per-task execution) sends a minimal 3-line prompt to `claude --print -p` worker subprocesses with no behavioral instructions, judges completion by CLI exit code only (exit 0 = pass via `empirical_gate_v1` in `classifiers.py`), and has no semantic QA. The system architect states there is "a very extensive system in place" for task verification. This investigation will definitively resolve whether that verification exists in a layer not yet examined (e.g., inherited CLAUDE.md governance, post-task hooks, pipeline gate systems) or whether this is a real architectural gap.

The investigation uses 8 parallel codebase research agents, 2 web research agents, 6 synthesis agents, and 3 QA gate checkpoints to produce a comprehensive research report at `.dev/tasks/to-do/TASK-RESEARCH-20260403-sprint-task-exec/RESEARCH-REPORT-sprint-task-execution.md`.

## Key Objectives

The following objectives MUST be achieved by this task:

1. **Trace the complete per-task execution path:** From `execute_sprint()` through `_parse_phase_tasks()` to `_run_task_subprocess()`, documenting exactly what prompt, context, and governance each worker subprocess receives.
2. **Investigate all potential verification layers:** Post-task hooks (wiring gate, anti-instinct gate), pipeline QA systems, PM agent systems, worker session inheritance (CLAUDE.md, skills, settings), and design intent from v3.1/v3.2 specs.
3. **Determine definitively whether verification gaps are real or missed:** Produce evidence-based findings with file paths and line numbers for every claim, distinguishing between "exists and works," "exists but not wired," and "does not exist."
4. **Produce actionable recommendations:** Options analysis with effort/risk tradeoffs for closing any confirmed gaps, plus a specific implementation plan for the recommended approach.
