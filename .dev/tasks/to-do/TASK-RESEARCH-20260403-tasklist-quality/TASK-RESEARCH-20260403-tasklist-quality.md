# Investigate Why TDD+PRD Enriched Input Produces Fewer Tasklist Tasks

## Task Overview

This task conducts a technical investigation into a counterintuitive behavior in the tasklist generation pipeline: when provided with 4.1x richer input (1,282 lines of TDD+PRD content vs 312 lines of spec-only), the pipeline produces 49% fewer tasks (44 vs 87) and 40% fewer phases (3 vs 5). The investigation spans six research areas -- protocol diff analysis, tasklist prompt analysis, roadmap phase structure comparison, task decomposition comparison, roadmap prompt analysis, and context window analysis -- to determine root cause and recommend specific code/prompt changes.

The investigation follows the tech-research skill's multi-phase pattern: parallel codebase research agents, analyst + QA verification gate, parallel synthesis agents, synthesis QA gate, report assembly with structural and qualitative QA, and user presentation.

## Key Objectives

The following objectives MUST be achieved by this task:

1. **Root Cause Identification:** Determine exactly which component(s) cause the task count regression -- roadmap generation (fewer phases/items), tasklist prompt instructions (suppressing decomposition), skill protocol changes (different decomposition rules), or context window saturation.
2. **Hypothesis Evaluation:** Evaluate all five hypotheses (H1-H5) from the research prompt with evidence-based verdicts citing specific file paths and line numbers.
3. **Fix Specification:** Produce specific, actionable code or prompt changes needed to fix the decomposition regression, identifying exact files, functions, and line ranges to modify.
4. **Structured Research Report:** Deliver a complete 10-section research report at `.dev/tasks/to-do/TASK-RESEARCH-20260403-tasklist-quality/RESEARCH-REPORT-tasklist-quality.md` following the tech-research skill's report structure.
