## 4. PRD Impact Assessment

### Positive Effects

1. **Obligation discharge (TDD)**: Undischarged obligations dropped from 5 to 1 (80% reduction). PRD context helps the roadmap generator produce fewer skeleton/stub placeholders, resulting in more complete phase descriptions.

2. **Extraction richness**: PRD enrichment substantially increased the number of extracted non-functional requirements (+56% total reqs for TDD, +50% for Spec), risks (+133% for both), and operational/migration items (+350-400% for TDD). The generator identified more components, dependencies, and migration concerns when PRD context was available.

3. **Multi-input state tracking**: The new pipeline correctly records `prd_file`, `tdd_file`, and `input_type` in the roadmap state file, enabling downstream consumers to understand the input provenance.

### Negative Effects

1. **Fingerprint coverage regression**: Coverage decreased in both pipelines (TDD: 0.76 to 0.73; Spec: 0.72 to 0.67). The PRD-enriched roadmap uses different terminology that misses some spec-specific fingerprints. In the Spec pipeline, the regression was more pronounced (-0.05 vs -0.03).

2. **Pipeline duration**: PRD enrichment increased total pipeline duration by 82% (TDD) and 51% (Spec). The extraction step was notably longer for TDD+PRD (~327s vs ~159s), and the generation steps also took longer.

3. **Retry instability**: The TDD+PRD run required a retry for haiku-architect generation (attempt 2), which did not occur in any other run. This may indicate that the larger extraction context from PRD enrichment occasionally exceeds model capacity for smaller models.

### Neutral Effects

1. **Contract coverage**: Completely unchanged across all runs. The uncovered contracts relate to structural pattern references (strategy_pattern, middleware_chain) that are independent of PRD context.

2. **Gate outcomes**: All 4 runs FAIL the anti-instinct gate. PRD enrichment does not change the final gate determination.

3. **Wiring verification**: All 4 runs PASS identically.

### Overall Assessment

PRD enrichment improves roadmap content quality (fewer stubs, richer extraction) at the cost of slightly reduced fingerprint fidelity and significantly increased pipeline duration. The trade-off is favorable for roadmap completeness but unfavorable for anti-instinct gate passage.

**Recommendation**: The fingerprint coverage regression warrants investigation. The anti-instinct scanner should consider whether PRD-introduced synonyms (e.g., using product-level terms instead of spec-level technical terms) should count as equivalent fingerprints. A synonym-aware fingerprint matcher could resolve this regression while preserving the extraction quality benefits.
