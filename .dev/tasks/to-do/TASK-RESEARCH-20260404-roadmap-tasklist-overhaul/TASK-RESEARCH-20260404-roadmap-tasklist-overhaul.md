# Deep Technical Investigation: Roadmap & Tasklist Generation Architecture Overhaul

## Task Overview

This task conducts a deep technical investigation of the Roadmap and Tasklist Generation Architecture in the SuperClaude pipeline. The investigation maps the complete end-to-end pipeline (every step, file, prompt, gate), identifies where granularity is destroyed during processing, assesses the one-shot output architecture that causes token-limit truncation, and produces a structured implementation plan to replace the current extraction-based approach with a template-driven, incremental-write architecture. The final report will be written to `.dev/tasks/to-do/TASK-RESEARCH-20260404-roadmap-tasklist-overhaul/RESEARCH-REPORT-roadmap-tasklist-overhaul.md`.

The current pipeline has three structural failures motivating this investigation: (1) extraction destroys granularity by summarizing 876-line TDDs into approximately 50 entities, (2) the one-shot output architecture hits token limits and truncates large outputs, and (3) the absence of output templates leads to inconsistent formats. These failures caused the TDD+PRD pipeline to produce 44 tasks versus the spec pipeline's 87 tasks from much less input.

## Key Objectives

The following objectives MUST be achieved by this task:

1. **Complete Pipeline Map:** Produce a detailed trace of every step in the roadmap and tasklist pipelines from CLI invocation to final output, including all prompt functions, gates, and execution mechanics.
2. **Granularity Loss Analysis:** Identify every point where granularity is destroyed or reduced, with specific evidence from prompt functions, extraction steps, and output capture mechanisms.
3. **One-Shot Architecture Assessment:** Analyze the ClaudeProcess output capture mechanism, token limits, and feasibility of incremental writing alternatives.
4. **Template Pattern Analysis:** Examine existing template patterns (TDD, PRD, MDTM) and design principles for roadmap/tasklist output templates.
5. **Implementation Plan:** Produce a specific, actionable implementation plan with file paths, function names, and integration points for replacing the current architecture.
