# Technical Research: PRD as Supplementary Pipeline Input

## Task Overview

Investigate how to add `--prd-file` as a supplementary input to both the roadmap and tasklist CLI pipelines, following the established TDD integration pattern. The roadmap pipeline currently loses PRD business/product context during TDD authoring -- the TDD skill extracts only 5 of approximately 28 PRD sections, and the roadmap extraction pipeline has zero steps for business rationale, personas, stakeholder needs, or success metrics. This investigation will determine exactly where and how PRD content should enrich the pipeline prompts, what supplementary validation blocks to add, and what skill/reference layer updates are needed.

The final deliverable is a structured research report at `.dev/tasks/to-do/TASK-RESEARCH-20260327-prd-pipeline/RESEARCH-REPORT-prd-pipeline-integration.md` containing findings, gap analysis, options, and a concrete implementation plan executable as a single task file.

## Key Objectives

- Map all integration points where `--prd-file` connects in roadmap CLI (commands, models, executor)
- Map all integration points where `--prd-file` connects in tasklist CLI (commands, models, executor, prompts)
- Determine which of the 10 roadmap prompt builders benefit from PRD enrichment and draft supplementary blocks
- Analyze which PRD sections are most pipeline-relevant and what content survives vs. is lost in TDD extraction
- Identify skill/reference layer updates needed (extraction-pipeline.md, scoring.md, SKILL.md, spec-panel.md)
- Research external patterns for PRD-driven roadmapping and value-based phasing
- Produce an actionable implementation plan with specific file paths, function names, and code patterns
