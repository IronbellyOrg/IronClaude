# Implement Option 3: TDD Template + Pipeline Upgrades for PRD/TDD/Spec Pipeline

## Task Overview

This task implements Option 3 from the TASK-RESEARCH-20260324-001 investigation, which upgrades the IronClaude PRD/TDD/Spec pipeline so that the TDD becomes the single source of truth feeding all downstream tools. The implementation modifies 6 files across 5 phases: adding pipeline frontmatter to the TDD template, upgrading sc:roadmap's extraction pipeline and scoring formula to consume TDD sections, upgrading sc:spec-panel to accept TDD input and produce scoped release specs, implementing sc:tasklist's declared-but-unimplemented `--spec` flag, and improving the PRD-to-TDD handoff in the tdd skill.

The research was verified by 6 parallel research agents, rf-analyst, and rf-qa (3 fix cycles). All instructions are based on code-verified findings (not documentation assumptions). Five CODE-CONTRADICTED claims were identified and corrected during research. The implementation plan (Section 8 of the research report) was QA-validated with all verdicts PASS.

- Add TDD input detection, release scoping, and scoped release spec output to sc:spec-panel

- Add a PRD Extraction Agent Prompt, update the Synthesis Mapping Table, add synthesis rules 12-13, and add QA gate item 13 to the tdd skill
