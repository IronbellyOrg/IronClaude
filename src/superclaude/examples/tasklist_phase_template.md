# Tasklist Phase Template

> **Usage**: This template defines the output structure for each phase file in the
> tasklist bundle generated by `/sc:tasklist`. All `{{SC_PLACEHOLDER:*}}` sentinels
> must be replaced during generation.
>
> **Sentinel self-check**: After generation, verify zero remaining sentinels:
> `grep -c '{{SC_PLACEHOLDER:' <output-file>` should return 0.
>
> **File naming**: Phase files MUST use `phase-N-tasklist.md` convention.
>
> **Content boundary**: Phase files contain ONLY tasks belonging to that phase.
> No cross-phase metadata, no registries, no global templates. Those live in
> `tasklist-index.md`.
